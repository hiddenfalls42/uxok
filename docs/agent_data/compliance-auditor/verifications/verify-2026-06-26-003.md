## Method

For each swept file I read the full page and checked, against `_base.py` ground truth:

1. **Runnability** — every `super().__init__(...)` passes only valid keyword args;
   every plugin instantiation matches its now-coreless constructor signature.
2. **Orphans** — no leftover `core` parameter, no method body using a removed `core`
   local, no stale prose telling the reader to pass `core` to a constructor.
3. **Import integrity** — every page that calls `Core()` imports `Core`; every page
   that dropped a `Core` annotation has no now-unused `Core` import.
4. **No RFC leaks** introduced by the sweep; guide meaning/steps preserved.

Two independent confirmatory greps over the 11 files (distinct from the
caller-established global grep):

- Orphan patterns `def __init__(self, core` / `, core[,):]` / `(core)` / `core: Core`
  / `super().__init__(core` → **zero matches** (exit 1).
- RFC literals `RFC ?[0-9]` / `§[0-9]` → **zero matches** (exit 1).
- All `\bCore\b` hits classified: each is an `import Core`, a `Core(...)` host
  instantiation, or a prose mention of the `Core` class. No stray annotations.

Word counts and the global pattern-absence grep were established by the caller and
taken as given. The 4 new secure-capabilities pages and the `AdmissionResult` field
names (verify-2026-06-26-002) were out of scope and not re-checked.
