#### M-001 — `docs/manifests/` is unexcluded internal content in the build tree
**File:** `docs/manifests/` (`API.md`, `KERNEL_ARCHITECTURE.md`, `FRAMEWORK_PHILOSOPHY.md`)
**Signal:** `Structural non-conformance`

`mkdocs.yml` `exclude_docs` lists `uxok/`, `agent_data/`, and `scratchboard/` but **not** `manifests/`. The three manifest files are constitutional/internal developer documents, not Diátaxis pages, and nothing in `gen_ref_pages.py` adds them to the generated `SUMMARY.md`. They will therefore be discovered by MkDocs as pages outside the nav. With `validation.nav.omitted_files: warn` they emit a WARNING, and `mkdocs build --strict` promotes every warning to an error — so the documented pre-merge gate (`mkdocs build --strict`) is at risk of failing on these orphans. The blueprint also forbids non-public content living loose in the public Diátaxis tree. Fix is an `exclude_docs` entry (`manifests/`), verified against the real `mkdocs.yml`.

**Doc ref:** `mkdocs.yml:10-13` (exclude_docs block); `docs/manifests/`

#### M-002 — Tutorial describes `self.hook()` as returning a list without `await`
**File:** `docs/tutorials/getting-started.md:206-208`
**Signal:** `Dangerous inaccuracy — version mismatch`

The Hooks section states: "Call `self.hook(name, *args)` to execute all registered handlers and collect their results as a list." The `hook` convenience method delegates to `HookSystem.execute`, which is `async` (`src/uxok/hooks/_system.py:146`); the convenience wrapper returns that coroutine directly (`src/uxok/plugin/_base.py:207`). A reader following the tutorial verbatim gets a coroutine object, not a list, and only sees results after `await`. The how-to page is correct here (`how-to-execute-hooks.md` uses `await self.hook(...)`), which makes the tutorial the outlier. Reword to show `results = await self.hook(...)`.

**Code ref:** `src/uxok/plugin/_base.py:165-215`, `src/uxok/hooks/_system.py:146-175`
**Doc ref:** `docs/tutorials/getting-started.md:206-213`

#### L-002 — Hook system opens by calling hooks "synchronous"
**Signal:** `Voice non-conformance`

`docs/explanation/hook-system.md:3` opens: "**Hooks** are synchronous extension points." Hook handlers are `async` and are awaited (`HookSystem.execute` is a coroutine; `await self._execute_hook_safe(...)` at `src/uxok/hooks/_system.py:211`), and the caller must `await self.hook(...)`. The intended meaning is "blocking/serial in the caller's task," which the body states correctly ("run every registered handler ... within the caller's own asyncio task ... The caller waits"). The opener's word choice is imprecise enough that a reader could infer hooks are non-async. Voice finding (Low) — the body recovers the meaning; tighten the first sentence to "serialized" or "blocking" to avoid the collision with Python's `sync`/`async` sense.

**Code ref:** `src/uxok/hooks/_system.py:146-215`
**Doc ref:** `docs/explanation/hook-system.md:3-7`

#### L-003 — Only one tutorial exists; `tutorials/` is thin for a "Tutorials" section
**Signal:** `Structural non-conformance`

`docs/tutorials/` contains a single page (`getting-started.md`) plus its index. The blueprint frames tutorials as a section ("`getting-started.md` is the first tutorial and comes first in nav"), implying a learning path of more than one. Either add a second tutorial (e.g. a hot-reload walkthrough, which the how-to already nearly covers in narrative form) or soften the index framing.

#### L-005 — `event-system.md` glob claim is correct but worth a doc-level note on `fnmatch` dot behavior
**Signal:** `Version mismatch (minor)`

`docs/explanation/event-system.md` states `cache.*` matches `cache.miss.deep` because `*` "matches zero or more characters including dot separators." This is accurate for Python `fnmatch` (the subscription matcher), and the tutorial's separate claim that `@event("counter.*")` "matches any event whose name starts with `counter.`" is consistent. No inaccuracy — but the two pages describe the same wildcard with slightly different precision ("starts with" vs the fuller dot-spanning rule). Align the tutorial wording to the explanation page's more precise statement so a reader does not infer single-segment matching. Low.

**Doc ref:** `docs/explanation/event-system.md` (Subscription patterns); `docs/tutorials/getting-started.md` (Events section)
